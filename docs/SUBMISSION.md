## 1. 심사 포지셔닝 3줄 (결과보고서 서두에 그대로 사용)

1. **규정이 권장하는 '생태계 소프트웨어'다.** 운영규정 [별표 2]는 "'AI 모델과의 연동 및
   개발 보조 생태계' 자체를 활성화하기 위한 **라이브러리**, 커넥터, 플러그인 형태의
   소프트웨어 개발 및 출품은 **적극 허용하며 권장함**"이라고 명시한다. 본 출품작은 최종
   사용자 앱이 아니라, 어떤 Android 앱에든 `implementation` 한 줄로 완전 오프라인 RAG 를
   넣는 **재사용 가능한 Kotlin 라이브러리(AAR)** 다. 데모 앱은 이 라이브러리의 소비자 예제다.
2. **바닥까지 이해하고, 측정으로 증명한다.** 벡터 인덱스(HNSW 포함)와 SIMD 커널(NEON/SSE2)을
   직접 구현했고, 재현율은 레퍼런스(hnswlib)와 동일 데이터에서 ±0.02 이내 동률임을 교차
   검증했다(운영규정 제9조⑤의 'AI 작성 코드 이해도' 조항에 대한 정면 대응).
3. **[별표 2] '독립 구동 가능성' 요건의 만점 답안.** 임베딩·검색·답변 생성 전 과정이
   기기 안에서 끝난다 — 외부 API 호출 0회. 한국어 문서·한국어 질의로 시연한다.

## 2.5 보고서용 의사결정 서사 — "왜 브루트포스가 기본값인가"

> 모바일 개인 문서 규모(수천~수만 청크)에서 SIMD 브루트포스는 1.7ms/질의(2만 청크)로 충분히
> 빠르다. 더 중요한 것: **docId 필터링이 스캔 루프 내부에서 정확도 손실 0**으로 동작하고,
> **삭제(removeDocument)가 연속 메모리풀 swap-remove 로 무손실**이다. 필터드 서치의 recall
> 붕괴와 노드 삭제가 알려진 난제인 ANN(HNSW) 계열과 대비되는 지점이다. 우리는 HNSW 를
> 직접 구현해 hnswlib 와 동률임을 검증한 뒤에도(→기술 역량 증빙) 기본값은 브루트포스로
> 남겼다 — 단순해서가 아니라, 이 규모에서 옳아서다. HNSW 는 수만 청크 이상용 옵션이다.

필터 구현 디테일(질문 대비): 허용 docId 집합을 id 인덱스 ByteArray 마스크로 변환해
JNI 경계를 **배열 1회**로 건넌다 — 후보별 Kotlin 콜백 업콜(µs × 청크 수)을 원천 차단.

## 3. 보고서용 핵심 수치 (전부 재현 가능)

- SIMD 내적: 스칼라 대비 **4.1×** (43.2→10.6ms / 2천만 호출, x86_64 SSE2; arm64 는 NEON 경로)
- 검색 지연(무작위 최악 조건, dim 384): 브루트포스 1.67ms@20k · 7.06ms@50k,
  HNSW **0.43ms@20k** (ef=100)
- 브루트 검색 전체 개선: 8.79 → 1.68 ms/질의 (**5.2×**, SIMD+k-힙)
- HNSW 재현율: hnswlib 와 ef 100~800 전 구간 **±0.02 이내 동률** (0.352/0.565/0.779/0.923
  vs 0.365/0.582/0.793/0.935) — 자체 구현 정확성의 외부 기준 검증
- JNI 경계 정직성: 질의 복사 1.5KB ≈ 무시 가능, C++ add 2~16µs/벡터 —
  인덱싱 병목은 임베딩 추론(ms/청크)이므로 배치 JNI 는 측정상 불필요 판단
- 스냅샷 복원: 재임베딩 없이 즉시 로드 (`RagEngine.save/load`, 데모 앱 버튼으로 시연)

## 4. 시연영상 3분 대본 (장면 = 강점 1:1 매핑)

**[0:00–0:20] 오프닝 — 문제 제기**
- 화면: 슬라이드 1장. "사내 문서를 ChatGPT 에 붙여넣어도 될까?" → "문서가 기기를

**[0:20–1:30] 장면 1 — 비행기 모드 데모 (강점: 독립 구동 + 한국어)**
- 액션: 상태바에 **비행기 모드 아이콘 보이게** → 데모 앱 실행 → 상태줄(임베더 e5·LLM ✓)
  잠깐 클로즈업 → "샘플 문서 4개 넣기" → 한국어 질문 "김치는 어떤 음식이야?" →
  **답변이 글자 단위로 스트리밍** + 근거 문서(유사도) 표시 → 처리시간(ms) 클로즈업
- 이어서: "저장" → 앱 강제 종료 → 재실행 → "불러오기" → "복원 N ms — 재인덱싱 없음" 표시
- 멘트 포인트: 전 과정 네트워크 0회 / 근거 표시로 환각 방지 / 재시작에도 즉시 복원.

## 4.5 결선 카드 (제출 후 · 데모 안정이 전제 — 지금은 착수 금지)

1. **SQ8 int8 스칼라 양자화** (1순위): 벡터당 1536B→~388B(4배↓, 5만 청크 75→19MB).
   NEON `vmull_s8`+`vpadalq_s16` 기본, ARMv8.2 dotprod 기기는 `SDOT` 런타임 분기(getauxval).
   기대 산출물: 메모리 4배↓ / 속도 / recall 곡선 3종 벤치 — "벤치마크 = 제2의 제품" 헤드라인.
2. 구조 인지 청킹: 마크다운 헤더·코드블록 경계 + 한국어 종결어미("다.", "요.") 인식.
3. 하이브리드 검색: 문자 bigram BM25 **자체 구현**(mecab 계열은 라이선스/NDK 비용으로 배제) + RRF 융합.

**추가하지 않기로 확정** (사유 포함 — 심사 질의 대비):
- 크로스인코더 리랭커(bge-reranker-v2-m3): 568M 파라미터 — Qwen 1.5B 위 메모리 예산 초과
- Qualcomm QNN/Hexagon EP: 독점 SDK 재배포 불가 — Apache-2.0 라이브러리에 부적합
- NNAPI EP: Android 15+ deprecated
- llama.cpp Vulkan 오프로드: Adreno/Mali 편차 커서 결선 멘토링 단계 소재
